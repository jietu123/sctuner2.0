# Figure 1（主文）推荐结构：4 个面板（A–D）

这张图的定位：**Mismatch 处理（filter / rescue / relabel）带来可量化的真值逼近，并且重分配是可解释、可校验的。**

------

## Panel A（必须有）：全局性能对照（Baseline vs Route2 vs Truth）

**图式**：paired dumbbell / slopegraph（不是柱状图）
**横轴**：多个指标（建议至少 5 个）
**纵轴**：指标值（每个指标一列）
**元素**：

- Baseline 一个点
- Route2 一个点
- 用线连起来（同 seed / 同场景配对）
- 旁边标注 Δ（Route2 - baseline）

**指标建议（按你们项目最“打脸审稿人”的顺序）**：

1. **L1_mean ↓**（越小越好）
2. **JS_mean ↓**
3. **corr_mean ↑**
4. **missing_leak_rate ↓（理想=0）**（如果该场景有 missing type）
5. **extra_pred_fraction ↓**
6. （可选）acc@1/3/5 ↑（如果你们 Stage5/Stage7能算）

**为什么它强**：
这一个面板直接回答：“Route2 有没有更接近真值？”——**没有这个面板，主图证据就是不完整的。**

**实现要点（给 Codex）**：

- 支持单 seed：就画一个配对线 + 标注数值
- 支持多 seed：每个 seed 一条淡线 + 叠加 mean/median 粗线/点（论文范儿）

------

## Panel B（必须有）：spot 级误差分布（“不是平均数蒙人”）

**图式**：ECDF / violin（推荐 ECDF，更硬）
**核心变量**：对每个 spot 计算误差改善量
$$
[
\Delta err(spot)=err_{route2}(spot)-err_{baseline}(spot)
]
$$
其中 err 可以是：

- spot 上类型分布的 **L1** 或 **JS**（建议 L1，直观）
- 或者 “topK 类型集合一致度” 也行，但 L1/JS最标准

**画法**：

- 横轴：Δerr（负数=变好）
- 纵轴：累计比例（ECDF）
- 两条曲线：Baseline（作为 0 参考） vs Route2 的 Δerr 分布（或直接画 Route2 的 Δerr ECDF）

**你要的“强语气结论模板”**（写在图注/正文都行）：

- 如果 ECDF 大量落在 0 左侧：**“Route2 在大多数 spot 上降低误差，不是少数 spot 的偶然收益。”**
- 如果只是少量 spot 极端改善：那就要诚实承认“收益集中”，但至少解释清楚。

------

## Panel C（强建议）：关键类型的“真值校准图”（回答你刚问的“广域补偿到底对不对”）

你现在最大困惑就是：**广域小幅补偿**是不是“不准确”。
结论同样很硬：**光看 delta map 你永远无法下判定**，必须用 truth 做校准。

**图式**：

### 方案 ：Truth vs Pred scatter（每个 spot 一个点）

对“争议最大的类型”（比如 CD8，或者被 mismatch 处理影响最大的那几个 type）分别画：

- 左：Baseline：pred_fraction vs truth_fraction
- 右：Route2：pred_fraction vs truth_fraction
  加 y=x 参考线，标注 corr、RMSE 或 slope。

**你要的判定就来自这里**：

- 如果 Route2 的点云更贴近 y=x、corr↑、RMSE↓：
  **广域补偿 = 正确纠偏（把不该是 CD8 的 mass 拿掉并分回合理类型）**
- 如果 Route2 更散、更偏离：
  **广域补偿 = “平均摊派式瞎补偿”】【必须否定】
